# CLAUDE.md — Talon Outreach Pipeline (รันคนเดียว)

Context สำหรับ Claude Code — **อ่านก่อนทุก session**

Repo นี้คือสายการผลิต **lead → demo → เสนอขาย** ที่รันโดยคนเดียว เพื่อ **พิสูจน์ว่า flow นี้ขายได้จริง** ก่อน scale
ทุกครั้งที่เจอลูกค้าเป้าหมาย: ดึงข้อมูล → สร้างเว็บ demo → เขียนเมลเสนอขาย → คิดราคา+แผน → ส่ง → เก็บสะสมที่นี่

> คนละ repo กับการรื้อ owldayhouse.com — อันนั้น = เว็บบริษัท, **อันนี้ = เครื่องมือหาลูกค้า**

---

## 🔒 กฎเหล็ก (ห้ามข้าม)

1. **ซื่อสัตย์ ไม่หลอก (NON-SCAM)** — เมลทุกฉบับโปร่งใส บอกตัวตนจริง บริษัทจริง ว่าเป็น demo ฟรีที่เราทำเอง ไม่อ้างเป็น Google ไม่ขู่ ไม่เร่งปลอม ให้ทางปฏิเสธชัด *(ดูหัวข้อ NON-SCAM ด้านล่าง)*
2. **ทำทีละ business** — แต่ละรายได้ครบชุด: เว็บ + เมล + ราคา/แผน เก็บคู่กันในโฟลเดอร์เดียว
3. **ใช้ Places API เท่านั้น ไม่ scrape Google Maps**
4. **PDPA** — บอกที่มาข้อมูล (Google สาธารณะ) + มีทางขอเลิกรับ
5. **คิดราคา + แผนใหม่ทุกครั้ง** ตามธุรกิจนั้น ไม่ก๊อปยกชุด
6. **อย่าแต่งข้อมูลที่ไม่รู้จริง** (เมนู ราคา คำโฆษณาเกินจริง) — ไม่มีก็เว้น/ใส่ placeholder

## 📁 โครงสร้าง repo (สะสมที่นี่)

```
talon-pipeline/
├── CLAUDE.md                  ← ไฟล์นี้
├── .env                       ← GOOGLE_PLACES_KEY (ห้าม commit)
├── scripts/
│   └── places-fetch.js        ← ชื่อร้าน/ลิงก์ → ข้อมูล → brief.md
├── templates/
│   └── brief.template.md      ← แม่แบบข้อมูลลูกค้า
└── leads/
    ├── INDEX.md               ← ตารางสถานะลีดทั้งหมด (pipeline)
    └── <ชื่อร้าน-slug>/
        ├── brief.md           ← ข้อมูล (Google + เติมมือ)
        ├── dist/index.html    ← เว็บ demo (static)
        ├── email.md           ← เมลเสนอขาย (คู่กับเว็บ)
        └── offer.md           ← ราคา + แผนเสนอขาย (เฉพาะรายนี้)
```

## ⚙️ Setup ครั้งเดียว
- Node ≥18, Claude Code
- ใส่ `GOOGLE_PLACES_KEY` ใน `.env`
- (ภายหลัง) Cloudflare Pages ไว้ host demo ให้ได้ลิงก์ · Resend ถ้าจะ auto ส่งเมล (ช่วงแรกส่งมือก่อน)

## 🔄 Workflow — ทำทุกครั้งที่เจอลูกค้า

เมื่อผู้ใช้สั่ง **"สร้าง demo สำหรับ &lt;ชื่อร้าน + ย่าน หรือ ลิงก์ Google&gt;"** ให้ทำตามลำดับนี้แล้วสรุปผล:

### 1) ดึงข้อมูล → `brief.md`
- รัน `scripts/places-fetch.js` ด้วย **ชื่อร้าน + ย่าน** (Text Search ชัวร์กว่าลิงก์สั้น)
- ได้: ชื่อ · ดาว · ข้อความรีวิว · เบอร์ · ที่อยู่ · เวลาเปิด · รูป → เขียน `leads/<slug>/brief.md` จาก `templates/brief.template.md`
- ช่องที่ Google **ไม่ให้** (FB Messenger, LINE, สี/มู้ด, เมนูที่คัด, รูปคุณภาพดี) → ทำเครื่องหมาย `⬜ ต้องเติมมือ` ให้ผู้ใช้กรอก
- ถ้าร้านไม่อยู่ใน Google → ข้ามไปกรอก brief มือล้วน

### 2) สร้างเว็บ demo → `dist/index.html`
- อ่าน `brief.md` → เว็บโปรไฟล์ static **ไฟล์เดียวจบ** (ไทย, mobile-first, โหลดไว, SEO meta + OG, ปุ่มติดต่อ) ลง `leads/<slug>/dist/`
- ใช้รีวิวจริงเป็นวัตถุดิบหาจุดขาย — **สรุปใจความเอง ห้ามวางคำรีวิวทั้งดุ้น**
- ปุ่มติดต่อ: ถ้ายังไม่รู้ FB/LINE จริง ใส่เบอร์จาก Google + หมายเหตุให้เติม · **ไม่มีฟอร์มเมล/backend**

### 3) เขียนเมลเสนอขาย → `email.md` (คู่กับเว็บ)
- ตามกฎ **NON-SCAM** ด้านล่าง · เขียนใหม่ให้เข้ากับร้านนั้นทุกครั้ง
- เก็บ `leads/<slug>/email.md`

### 4) คิดราคา + แผน → `offer.md` (เฉพาะรายนี้)
- เลือก tier เหมาะกับธุรกิจ + add-on ที่เกี่ยวข้อง + ออปชันดูแลรายเดือน *(ดู "ราคา/แผน")*
- เก็บ `leads/<slug>/offer.md`

### 5) อัปเดต `leads/INDEX.md`
- เพิ่มแถว: `ชื่อร้าน | ประเภท | วันที่ gen | ราคาเสนอ | สถานะ`
- สถานะไล่: `demo-ready → sent → replied → won / lost`

### (เมื่อจะส่ง)
- host `dist/` ขึ้น Cloudflare Pages → ได้ลิงก์ → ใส่ลิงก์ในเมล *(ส่งลิงก์ดีกว่าแนบไฟล์ HTML — คนระวังไฟล์แนบ)*
- **ส่งเมลจากอีเมลคุณเอง รีวิวก่อนส่ง** → อัปเดต INDEX เป็น `sent`
- ช่วงเริ่ม **ส่งมือทีละราย** — deliver ดีกว่า + ดูจริงใจกว่า bulk + เลี่ยงโดน spam filter

## ✉️ NON-SCAM — กฎการเขียนเมล (สำคัญที่สุด)

เมลคือ outreach ที่ไม่ได้ร้องขอ จึงต้อง **โปร่งใสและให้เกียรติ** เสมอ

**ต้องมี:**
- บอกชื่อจริง + **บริษัท อาวล์ เดย์ เฮ้าส์ จำกัด (จดทะเบียนจริง)** ตั้งแต่ต้น
- บอกตรง ๆ ว่าเป็น **demo ฟรีที่เราทำให้ดู ไม่ได้ถูกร้องขอ**
- บอกที่มา: **เห็นร้านใน Google Maps**
- ลิงก์ดู demo จริง + ราคาเริ่มต้นที่ชัด
- ทางปฏิเสธง่าย: *"ถ้าไม่สนใจ ไม่ต้องตอบก็ได้ครับ / แจ้งมาได้ จะไม่รบกวนอีก"*
- ลงชื่อ + ช่องทางติดต่อจริง

**ห้ามเด็ดขาด:**
- อ้างเป็น/เกี่ยวข้องกับ Google หรือหน่วยงานใด
- ขู่/หลอกว่าร้านจะเสียอันดับ / ถูกลบ / มีปัญหา ถ้าไม่ทำเว็บ
- สร้างความเร่งปลอม ("เหลือ 2 ที่", "ราคานี้วันนี้เท่านั้น") ถ้าไม่จริง
- หัวข้อเมลหลอกลวง ("Re:" ทั้งที่ไม่เคยคุย / "ใบแจ้งหนี้" / "บัญชีของคุณ")
- สัญญาเกินจริง ("ติดอันดับ 1 Google ใน 1 วัน")
- ขอข้อมูลส่วนตัว / เงิน / รหัส ในเมลแรก

**โทน:** ช่วยเหลือ มืออาชีพ ไม่กดดัน สั้น อ่านจบเร็ว

**โครงเมล (เขียนใหม่ให้เข้ากับร้านทุกครั้ง):**
1. ทักทาย + เราเป็นใคร (บริษัท)
2. เห็นร้านใน Google รีวิวดี (X★ จาก N รีวิว) สังเกตว่ายังไม่มีเว็บไซต์
3. เลยลองทำ **เว็บตัวอย่าง** ใส่ข้อมูลร้านให้ดู → [ลิงก์ demo]
4. ถ้าสนใจให้ดูแลต่อ เริ่มที่ [tier + ราคา] + ออปชัน
5. ไม่สนใจ ปฏิเสธได้ ไม่รบกวน + ลงชื่อ/ช่องทางจริง

## 🌐 เนื้อหาเว็บ demo
- รีวิวจริง = วัตถุดิบหาจุดขาย แต่ **สรุปใจความเอง** ไม่ก๊อปคำรีวิว
- ไม่รู้จริง อย่าแต่ง (เมนู/ราคา/รางวัล) — เว้นหรือใส่ placeholder ที่ลูกค้าเติมได้
- ทุกหน้า: `<title>` + meta description + OG + `lang="th"` + รูปมี alt

## 💰 ราคา/แผน (`offer.md`) — ต่อราย

อิงบันได: **เว็บโปรไฟล์ 3,900 · เว็บธุรกิจ 6,900 · Custom 15,000** + ดูแลรายเดือน

เลือก tier + add-on ตามประเภทร้าน:
- **ร้านอาหาร/คาเฟ่** → เมนูออนไลน์ · แกลเลอรี · แผนที่/รีวิว
- **คลินิก/สปา/ความงาม** → ระบบจอง · ภาพ before/after · รีวิว
- **อู่/บริการ** → ตารางราคา · นัดหมาย · แผนที่
- **ค้าปลีก** → แคตตาล็อก · ลิงก์ช้อป/LINE

แต่ละ `offer.md` มี: tier+ราคาที่แนะนำ · เหตุผลสั้น ๆ · add-on 1-2 ที่เกี่ยว · ออปชันดูแลรายเดือน · next step
> ดูแลรายเดือนคิดตามต้นทุนจริง — host static **ฟรีได้** (Cloudflare) จ่ายแค่โดเมน อย่าตั้งราคาขาดทุน
